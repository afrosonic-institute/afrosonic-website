# Afrosonic — MVP Website

The MVP website for **Afrosonic**, a Nigerian music heritage non-profit based in Abuja.

- **Public name:** Afrosonic
- **Legal name:** IPAHM — Initiative for Preservation and Archiving of Historical Music
- **Production domain:** afrosonic.org *(pending DNS migration from WordPress)*
- **Hosting:** Vercel (Hobby tier, auto-deploys from `main`)
- **Status:** v0.1.0 — initial scaffold from v0.app, static content live, content placeholders pending

---

## What's actually in this repo

This is **not** a typical Next.js application. It's a Next.js 16 scaffold (generated by [v0.app](https://v0.app)) wrapped around a hand-written static HTML site that lives in `/public/`.

```
Visitor → afrosonic.org → Next.js root page (app/page.tsx)
                       → redirects to /index.html
                       → served as a static file from /public/
```

In plain terms: the Next.js layer is a thin host. It does three jobs and three jobs only:

1. **Routes the root URL** to the static homepage (`app/page.tsx` redirects `/` → `/index.html`).
2. **Injects Vercel Analytics** in production (`app/layout.tsx`).
3. **Serves the favicons and metadata** declared in `app/layout.tsx`.

All real website content — markup, copy, styling, navigation — lives in `/public/`. To edit the site, edit the HTML and CSS files in that folder.

The `components/`, `hooks/`, and `lib/` folders contain a full shadcn/ui kit that came with the v0 scaffold. **None of these components are currently used** by the live site. They are kept in the repo for now in case they're useful for future React-based features (e.g. a store page).

## Local development

You have **two equally valid ways** to preview the site locally.

### Option A — Open the static files directly

Fastest. No install needed.

```bash
open public/index.html
```

Navigation between pages works because each link uses an absolute path (`/rawa.html` etc.), and most modern browsers resolve those against the file's directory when opened locally. If a link breaks, use Option B.

### Option B — Run the Next.js dev server

Required if you want the `/` → `/index.html` redirect, Vercel Analytics behaviour, or to test exactly what production serves.

```bash
pnpm install
pnpm dev
```

The site will be available at `http://localhost:3000`.

| Script         | What it does                                  |
| -------------- | --------------------------------------------- |
| `pnpm dev`     | Start the Next.js dev server                  |
| `pnpm build`   | Build for production                          |
| `pnpm start`   | Run the production build locally              |
| `pnpm lint`    | Run ESLint (no config currently shipped)      |

> **Note on Node version:** Next.js 16 + React 19 requires Node 18.18+ (Node 20 LTS recommended).

## Editing the site — quick guide

| If you want to…                                | Edit this                                            |
| ---------------------------------------------- | ---------------------------------------------------- |
| Change copy, layout, or markup on a page       | `public/<page>.html`                                 |
| Change colours, fonts, or section styling      | `public/styles.css`                                  |
| Add a new static page                          | New `public/<name>.html` + link from nav             |
| Change the homepage URL or redirect target     | `app/page.tsx`                                       |
| Change the site `<title>` default or favicons  | `app/layout.tsx`                                     |
| Change deploy behaviour                        | `next.config.mjs` or Vercel project settings        |

Do **not** put website content in `app/page.tsx`. The redirect is intentional — the static HTML is the source of truth.
